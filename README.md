# MuSS-17M

Utilities for syncing and preprocessing multi-view (multi-camera) behavioral videos.

This repo contains a lightweight Python pipeline under `Video_Preprocess/` that helps you:
- rename and normalize camera folder video filenames
- extract per-frame timestamps (cap_msec) to CSV
- compute cross-camera frame correspondences from user-provided sync points
- extract synchronized frames for downstream calibration / 3D pose estimation
- render montage videos for sanity-checking

## Folder layout (expected)

Most scripts assume a directory like:

```
<video_root>/
  Camera1/
    0.mp4
    1.mp4
    ...
  Camera2/
    0.mp4
    ...
  ...
  Camera6/
    0.mp4
  csv/                       # generated by 1_get_frame_info.py
```

## Quick start

Install deps (typical):

```bash
pip install opencv-python numpy matplotlib tqdm
```

Then run (examples):

```bash
cd Video_Preprocess

# 0) rename Camera*/ videos to 0.mp4, 1.mp4, ...
python 0_rename_folder.py /path/to/video_root

# 1) extract per-frame timestamp CSVs for selected videos
python 1_get_frame_info.py --root /path/to/video_root --videos 0 1 2

# 2) compute synchronized frame mapping (provide sync points)
python 2_sync_frames_finder.py \
  --dir /path/to/video_root \
  --video-name 0 \
  --sync-points "1=315,2=298,3=287,4=292,5=297,6=289"

# 3) extract synchronized frames
python 3_extract_frames.py --video-root /path/to/video_root --video-name 0

# 6) montage check from extracted frames
python 6_montage_sync_frames.py \
  --frames-root /path/to/video_root/frames/0.mp4/all \
  --output-video /path/to/video_root/check/montage_refCam1.mp4 \
  --reference-camera 1 \
  --fps 60
```

## Notes

- These scripts are designed to be path-portable: no hard-coded absolute paths.
- Some scripts optionally use `ffmpeg/ffprobe` if installed; otherwise they fall back to OpenCV where possible.
